## Model summary

**Context:** AI Agent / Operator → Cursor / agent-lsp; agent-lsp → Docker Engine,
Git remotes / mirrors.

**Containers** (inside agent-lsp): Caddy, MCP Server, Runtime Health Worker,
SQLite State, Git Worktree Store, LSP Runtime Containers.

**Components** (inside MCP Server): Scout Tools API, RuntimeHub,
LspClient / TcpTransport, TaskStore + ScoutWorker, GitService, DockerService,
StateStore.

**Code** (V1 atom edges `code→code`): `server.py`, `blast.py`, `deps.py`,
`runtime_hub.py`, `lsp_client.py`, `worker.py`, `task_bridge.py`, TaskStore
(Rust), agent-lsp-{git,docker,state}, mirrors.py, agent-lsp-runtime-worker.
Relationships mirror the import/call graph (server→hub→client, worker→TaskStore,
hub→docker/state, etc.).

### Local scout (this machine)

```bash
# HTTP MCP on :8765 (ALLOW_LOCAL + host pyright)
AGENT_LSP_ALLOW_LOCAL=1 \
AGENT_LSP_STATE=$PWD/.data/state \
AGENT_LSP_PROJECTS=$PWD/.data/projects \
AGENT_LSP_WORKSPACES=$PWD/.data/workspaces \
AGENT_LSP_CACHE=$PWD/.data/cache \
uv run python -c 'from agent_lsp.paths import ensure_data_dirs; from agent_lsp.server import mcp; ensure_data_dirs(); mcp.run(transport="http", host="127.0.0.1", port=8765)'

# In-process onboard + blast of this repo:
uv run python scripts/c4_local_onboard_blast.py
```

## How to edit

Use architect-c4 MCP tools (session + explicit `workspace_id` on checkout):

1. `create_session` → `create_project` → `checkout_workspace` (**must** pass
   non-null `workspace_id`)
2. `upsert_element` / `upsert_relationship` / `upsert_flow` / `upsert_adr`
3. `validate_model` → `get_view_links` with
   `base_url=https://architecture.runmcp.ru`

V1 atom canon allows relationship edges only between
`person` / `software_system` / `code` / `external`. Containers and components
nest via `parent_id`; use flows for dynamic interaction.
