```text
L1 person/system
  AI Agent, Operator → Cursor, agent-lsp → Docker, Git remotes

L2 containers (agent-lsp)
  Caddy | MCP Server | Runtime Health Worker | SQLite | Git store | LSP runtimes

L3 components
  MCP: Scout Tools, RuntimeHub, LspClient, TaskStore, Git, Docker, State
  + Caddyfile, SQLite schemas, git FS, LSP images, runtime-worker binary

L4 code (V1 atom edges: code↔code only)
  server/_with_lsp_client/_client_for/blast/deps
  RuntimeHub/SessionRuntime/ensure_container/mark_runtime_stale
  LspClient/TcpTransport/StdioTransport
  ScoutWorker/wake_worker/await_sqlite_task/TaskStore
  DockerService/BollardDockerAdapter
  GitService/GixGitAdapter/mirrors/paths
  StateStore + sessions/tasks schemas
  runtime-worker: main/reconcile_once/mark_runtime_stale/…
```

Container/component edges in the viewer are **rolled up from code→code**
relationships (V1 atom canon forbids direct container↔container edges).

### Note on `+…()` in code views

architect-c4 L4 always exports Mermaid `classDiagram`. When a code element's
`members` list is empty (the MCP `upsert_element` API does not accept members),
the exporter draws a placeholder `+…()` — **not** real methods of a Python
module. Prefer naming: real **classes** (`RuntimeHub`, `LspClient`),
**functions** (`blast_radius`, `_with_lsp_client`), not `*.py` as a class.

## Local scout

```bash
AGENT_LSP_ALLOW_LOCAL=1 \
AGENT_LSP_STATE=$PWD/.data/state \
AGENT_LSP_PROJECTS=$PWD/.data/projects \
AGENT_LSP_WORKSPACES=$PWD/.data/workspaces \
AGENT_LSP_CACHE=$PWD/.data/cache \
uv run python -c 'from agent_lsp.paths import ensure_data_dirs; from agent_lsp.server import mcp; ensure_data_dirs(); mcp.run(transport="http", host="127.0.0.1", port=8765)'

uv run python scripts/c4_local_onboard_blast.py
```
