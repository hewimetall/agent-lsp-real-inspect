## Model summary

**Context:** AI Agent / Operator → Cursor / agent-lsp; agent-lsp → Docker Engine,
Git remotes / mirrors.

**Containers** (inside agent-lsp): Caddy, MCP Server, Runtime Health Worker,
SQLite State, Git Worktree Store, LSP Runtime Containers.

**Components** (inside MCP Server): Scout Tools API, RuntimeHub,
LspClient / TcpTransport, TaskStore + ScoutWorker, GitService, DockerService,
StateStore.

## How to edit

Use architect-c4 MCP tools (session + explicit `workspace_id` on checkout):

1. `create_session` → `create_project` → `checkout_workspace` (**must** pass
   non-null `workspace_id`)
2. `upsert_element` / `upsert_relationship` / `upsert_flow` / `upsert_adr`
3. `validate_model` → `get_view_links` with
   `base_url=https://architecture.runmcp.ru`

V1 atom canon allows relationship edges only between
`person` / `software_system` / `code` / `external`. Containers and components
nest via `parent_id`; use flows for dynamic interaction.
